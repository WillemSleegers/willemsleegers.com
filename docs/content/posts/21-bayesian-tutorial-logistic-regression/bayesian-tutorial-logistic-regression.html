This post is about using brms to run a logistic regression. Logistic regression is different from the models from previous posts because it involves a dichotomous outcome measure. This is different from modeling means, which we have roughly been doing, so there are some new things to keep in mind here.

Run the following code to get started. Note that the data weâ€™ll be using is the titanic dataset. It contains several interesting columns such as whether or not the passenger survived, their sex, and their age.

Code 
library(tidyverse)
library(brms)
library(tidybayes)

data <- read_csv("titanic.csv")

theme_set(theme_minimal())

colors <- c("#93CFDB", "#1E466E")

Letâ€™s say that we want to model the proportion of survivors. We can do that by running a logistic regression in which we regress the outcome (survived: 0 or 1) on an intercept and specify that the outcome comes from a binomial family with a logit transformation. We use the get_prior() function to get a first grasp of which priors we need.

Code 
get_prior(survived ~ 1, data = data, family = binomial(link = "logit"))
prior class coef group resp dpar nlpar lb ub source
student_t(3, 0, 2.5) Intercept default

We only need a single prior for the intercept. Thatâ€™s easy enough, but the family and link argument are kinda new.

\[\displaylines{survived_i âˆ¼ Binomial(n, p_i) \\ logit(p_i) = \alpha}\]

Setting the priors

You might think that because weâ€™re modelling a proportion, the prior for the intercept should be a distribution of values constrained to range from 0 to 1. This is not the case because of the logit transformation. The logit link transforms the probability scale onto a linear scale that ranges from minus infinity to infinity, although the bulk of the values are around 0. We can show this in a graph.

Code 
df <- tibble(
  x = seq(0, 1, 0.001),
  logit = logit_scaled(x)
)
ggplot(df, aes(x = x, y = logit)) +
  geom_point() +
  labs(x = "p", y = "logit(p)")

Our prior needs to be set on the transformed values, not the original probability scale. brms sets a default prior of student_t(3, 0, 2.5). Letâ€™s take a look at what this looks like by running a model and sampling only from this prior.

Code 
fit_prior_default <- brm(
  survived ~ 1,
  family = binomial(link = "logit"),
  data = data,
  file = "model-prior-default.rds", 
  sample_prior = "only"
)

We get a warning that using the Bernoulli family might be more efficient because we only have 2 levels and that we need to specify the number of trials when using the Binomial family; letâ€™s keep this in mind when we run the code again. For now, letâ€™s plot the prior.

Default brms prior on the intercept in a logistic regression

The top graph shows the prior on the logit scale. The bulk of the values are somewhere between -10 and 10, concentrated around 0. This doesnâ€™t seem like a suitable default prior to me because if we convert these values to the probability scale, it turns out that the extreme values (around 0 and 1) are relatively more likely than other values. I would probably prefer a prior that is either more uniform or perhaps centered around .5, depending on the context. This is likely to be a weak prior though, so it will be easily overrun by data. Nevertheless, letâ€™s try out another prior and see if it produces a more uniform distribution on the probability scale. Letâ€™s also change the binomial() family to a bernoulli() family.
